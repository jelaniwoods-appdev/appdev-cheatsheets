Keyword
Description
Example
true, false
Conditional program execution: program starts with if branch, tries the elsif branches, and finishes with else branch (until one branch evaluates to else)
if, elsif, else
An if statement evaluates an expression, which returns either true or false. When true, Ruby executes the "code block" that follows. When the expression is false, Ruby returns nil. In this example, the string "Your condition was true!" will print because the condition number == 10 is true.
nil
nil is a special Ruby object used to represent an "empty" or "default" value. Many methods can return nil as a result. This happens when you ask for a value but that value is not available:
  • Array index out of bounds
  • Hash key or value doesn't exist
  • Undefined instance variables
  • Record not found in our database
||
The || (pronounced "or") operator is a logical operator which returns true if either of the expressions on left-hand side or right-hand side evaluate to true.
&&
The && operator (pronounced "and") is a logical operator in which evaluates to true only if both expressions on either side of && evaluates to true.
==
A logical operator in which compares the expressions on both sides and returns true is both values are the same. Otherwise, it returns false.
def
Defines a method. A method is a block of code that can easily be run by invoking the method name. A def requires a method name and an end to denote where the method ends. All methods must be called on some object.
return
A keyword that allows the developer to explicitly stop the execution flow of a method and return a specific value. Note: values that are returned are not printed by
class
A class is the blueprint from which individual objects are created. Classes allow us to write methods and attributes which can be reused by many objects.
